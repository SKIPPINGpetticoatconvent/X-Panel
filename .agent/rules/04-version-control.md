# 4. 版本控制规范

## Commit Message 规范

必须遵循 [Conventional Commits](https://www.conventionalcommits.org/zh-hans/)：

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修补 Bug |
| `docs` | 文档修改 |
| `style` | 代码格式修改（无运行影响） |
| `refactor` | 重构（非新增功能、非修 Bug） |
| `chore` | 构建/辅助工具变动 |

## 分支命名

- 功能开发: `feat/`
- 修复: `fix/`
